## Decisions

- [2026-06-30 19:15] Adopt backfill-from-transcript over proxy intercept or accepting loose attribution — three paths were offered: (a) add a Bedrock-specific intercept in the model path, (b) stop and accept two unconnected spans, (c) replay on-disk transcript JSONL to lapdog's existing `/claude/hooks/backfill_session` endpoint. Path (a) was rejected because `CLAUDE_CODE_USE_BEDROCK=1` routes through `bedrock-runtime.<region>.amazonaws.com`, not the Anthropic Messages API, making a general proxy intercept non-applicable and requiring model-path-specific Bedrock instrumentation instead. Path (c) won: it reuses the server-side converter, is Bedrock-agnostic, and uses JSONL transcripts that carry real timestamps (stream-json pipe carries none). This reversed a prior thread decision of "No proxy — parked as Phase 3." (46037a58 · turn ~25)

- [2026-06-30 19:20] Make terminal hook events synchronous, keep all others async — `Stop` and `SessionEnd` set `async: false`; every other event remains `async: true`. Rationale: `docker run --rm` reaps the container immediately on exit and fire-and-forget curl calls issued inside the container cannot be guaranteed to complete before the container is reaped. Terminal events must block to ensure delivery. (46037a58 · turn ~25)

- [2026-06-30 20:30] Accept cost span and hook agent span remaining in separate llmobs traces — single-trace unification via proxy bridging not pursued. Per-session cost attribution achieved through session_id extraction and backfill replay; the separation is by design given the Bedrock constraint. (46037a58 · turn ~80)

## Learnings

- [2026-06-30 19:00] Both bugs share a common root: LLM calls reach the proxy but the matching hooks session is absent — all affected sessions show `instrumented: false`, exactly one prompt recorded, and never close. Cost spans therefore have no session anchor and collapse into a sessionless bucket. (46037a58 · turn 2)

- [2026-06-30 19:01] Only `Stop` or `SessionEnd` events finalize root spans in lapdog; every other event leaves the span open — this alone accounts for Bug #1, sessions remaining indefinitely running. (46037a58 · turn ~8)

- [2026-06-30 19:05] The threads skill spawns separate `claude` processes inside containers as headless agents, not as subagents running within the parent session — session_id therefore does not flow through the parent's hook pipeline, explaining why cost spans arrive unattributed. (46037a58 · turn ~15)

- [2026-06-30 19:05] `docker run --rm` reaps the container immediately on process exit; async fire-and-forget curl calls issued from inside the container have no delivery guarantee before the reap. (46037a58 · turn ~15)

- [2026-06-30 19:12] `CLAUDE_CODE_USE_BEDROCK=1` routes all LLM calls through `bedrock-runtime.<region>.amazonaws.com` using SigV4-signed Converse/InvokeModel, not the Anthropic Messages API — the direct Anthropic API intercept expected for `claude -p --output-format stream-json` does not apply. (46037a58 · turn ~22)

- [2026-06-30 19:20] stream-json output carries no timestamps; on-disk transcript JSONL carries them — lapdog's backfill converter requires timestamps, so replaying from JSONL is the correct mechanism. (46037a58 · turn ~30)

- [2026-06-30 20:30] The `/claude/hooks/backfill_session` endpoint is idempotent — duplicate POSTs for the same session are accepted without error. (46037a58 · turn ~80)
